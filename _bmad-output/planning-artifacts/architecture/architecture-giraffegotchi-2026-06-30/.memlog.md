- (direction) Paradigm: layered — pure domain core (pet) -> rendering (ui) -> orchestration (main), driven by a single-threaded frame loop. Ratified from existing code; the native-test split already proves the core boundary.
- (decision) AD-1 [ADOPTED] Pure domain core stays hardware-free. Binds: pet + any extracted pure-logic module (e.g. solar/sky math). Prevents: Arduino/TFT includes leaking into logic, which breaks env:native. Rule: modules built under env:native include no Arduino/TFT/FS headers and carry Unity tests.
- (decision) AD-2 Layer dependency direction is one-way. Binds: all. Prevents: cycles / render reaching into orchestration / core depending on render. Rule: main -> {ui, pet} -> hw libs; pet depends on nothing; ui may use pet (Emotion, thresholds) but never main; no module depends upward.
- (decision) AD-3 Orchestration owns frame cadence + panel. Binds: main, ui, anim. Prevents: two modules pushing to the TFT out of order -> tearing. Rule: only the loop issues the top-level composite+push; render/anim modules expose draw fns taking a target surface (TFT_eSPI& / TFT_eSprite&), never own timing or call the top-level push.
- (decision) AD-4 [ADOPTED] Single compositing band owns the giraffe footprint (x85-235,y34-194). Prevents: half-drawn frames/flicker on the un-double-buffered panel. Rule: in-box content composites into skyBand then ONE atomic pushSprite/frame; out-of-box content draws direct with viewport pixel-clip; nothing draws direct inside the box.
- (decision) AD-5 [ASSUMPTION] Animation pose-ownership contract. Prevents: two animations mutating giraffeBuf -> wrong pose; ambiguous frame ownership. Rule: <=1 animation owns the pose buffer per frame (kick owns exclusively via setKickPose; others only compose foreground into the band). Extracted anim module exposes start(now)/tick(now)/compose(band) and never pushes to the panel. Needs Elliot ratification — constrains the anim-extraction design.
- (decision) AD-6 [ADOPTED] Single transparency-key source. Magenta 0xF81F is the global key, read at runtime from giraffeBuf[0]; never appears inside a silhouette. Prevents: sprite holes / divergent keying. Rule: render paths key off buf[0], not a second hardcoded literal; changing it means updating prep_sprite.py MAGENTA_RGB too.
- (decision) AD-7 [ADOPTED] restoreBg is the only erase. Prevents: flat fills that skip prop/band repaint -> trails/stale pixels. Rule: all erase-to-bg goes through restoreBg(x,y,w,h); never fillRect with a bg colour; never panel readback (readRect broken on this CYD).
- (decision) AD-8 [ADOPTED] Persistence = versioned NVS blob, throttled, restore-as-is. Binds: save module. Prevents: unversioned layout change corrupting reads; flash wear; time-aging on restore. Rule: PetSave fixed struct guarded by SAVE_MAGIC (bump on change); save <=1/5s and only when dirty; restore does not advance decay; only Pet stats + prank-death flag persist.
- (decision) AD-9 [ADOPTED] Config/secrets via build-flag injection. Binds: all. Prevents: secrets in source; build requiring .env. Rule: WiFi/lat/lon/TZ from .env via tools/load_env.py as -D flags; every injected flag has a #ifndef in-source fallback; no credentials committed.

- (direction) SCOPE CONFIRMED by Elliot: the refactor is NOT 'split the god files' — it is 'make animals swappable for the giraffe'. Choices: (1) runtime on-device swap, (2) animals may animate completely differently, (3) data-driven animation engine. pet.* stays untouched (already species-agnostic). - (direction) Paradigm reframed: layered firmware + a DATA-DRIVEN SPECIES REGISTRY. The giraffe becomes one descriptor in a registry; render/anim read the active descriptor, no hardcoded giraffe-isms remain in code.
- (decision) AD-11 Single source of species truth. Binds: render, anim, all. Prevents: giraffe-isms leaking back into code; two places defining an animal -> a swap misses one. Rule: everything species-specific (asset folder, geometry W/H/X/Y, anchor points mouth/sleep/daydream, animation set, capabilities) lives ONLY in the active Species descriptor. No /giraffe_* literal, no GIRAFFE_W, no anchor constant anywhere else. No compile-time giraffe default baked into render/anim.
- (decision) AD-12 Animations are data, played by one engine. Binds: anim. Prevents: per-animal animation code diverging; adding an animal requiring engine edits. Rule: an animation = data (frames, anchor ref, cadence, layer=pose|foreground-band) played by a species-agnostic generic engine. A signature behaviour not expressible as data (giraffe ball-kick) is a NAMED CAPABILITY HOOK the descriptor opts into; the engine calls it and the hook still obeys AD-4 (band ownership) + AD-5 (single pose writer). Adding a data-only animal touches no engine code.
- (decision) AD-13 Active species: one owner, persisted, atomic reload-on-swap. Binds: main, save, render. Prevents: half-swapped state (new geometry+old sprites); active-species desync across modules; losing selection on reboot. Rule: orchestration owns the single active-species pointer; a swap is atomic — reload the footprint's sprite buffers before the next composite, no frame straddles two species. Active species id persists via AD-8's versioned NVS blob and restores on boot. Others read active species through the owner (AD-10), never cache a copy.
- (decision) AD-14 Asset namespacing + flash budget. Binds: art pipeline, render, build. Prevents: sprite-name collisions across animals; silently overflowing the LittleFS partition. Rule: each species owns a folder /<species>/ with identical sprite filenames (happy.png, blink.png, ...); the descriptor names the folder, not individual paths. Total shipped sprite bytes must fit the LittleFS partition with headroom; the prep/build step reports the budget. (Forced by the runtime-swap choice.)
